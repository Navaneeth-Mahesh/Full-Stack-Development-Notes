# Web APIs

Web APIs (Application Programming Interfaces) are built-in browser features that allow JavaScript to interact with the browser, the user's device, and external services.

Without Web APIs, JavaScript can only perform calculations and manipulate variables. Web APIs extend JavaScript by providing access to browser capabilities such as:

- User Location
- Camera & Microphone
- Local Storage
- Drag & Drop
- Notifications
- Background Threads
- Clipboard
- Fetching Data
- Timers
- WebSockets
- And much more...

> **Important:** Web APIs are provided by the browser (Chrome, Firefox, Edge, Safari), **not by JavaScript itself**.

---

# How Web APIs Work

```
                User

                  │
                  ▼
        ---------------------
        |     Web Browser   |
        ---------------------
                  │
                  ▼
        ---------------------
        |    JavaScript     |
        ---------------------
                  │
                  ▼
        ---------------------
        |     Web APIs      |
        ---------------------
          │    │     │
          │    │     │
          ▼    ▼     ▼
     Storage Location Network
```

JavaScript requests something from a Web API, and the browser performs the task.

Example:

```javascript
navigator.geolocation.getCurrentPosition(showLocation);
```

JavaScript asks the browser:

> "Please tell me the user's location."

The browser:

- Requests permission
- Gets GPS/Wi-Fi/IP location
- Returns the result to JavaScript

# 1. Geolocation API

The Geolocation API allows websites to access the user's current location.

Common uses:

- Maps
- Food delivery apps
- Weather apps
- Ride booking
- Nearby stores

---

## How It Works

```
JavaScript
     │
     ▼
Browser asks permission
     │
     ▼
User clicks Allow
     │
     ▼
Browser gets location
(GPS / Wi-Fi / IP)
     │
     ▼
Coordinates returned
(latitude & longitude)
```

---

## Syntax

```javascript
navigator.geolocation.getCurrentPosition(successFunction);
```

Example:

```javascript
navigator.geolocation.getCurrentPosition((position) => {
    console.log(position.coords.latitude);
    console.log(position.coords.longitude);
});
```

---

## Important Methods

### Get Current Position

```javascript
navigator.geolocation.getCurrentPosition();
```

### Watch Position

Continuously tracks movement.

```javascript
navigator.geolocation.watchPosition();
```

---

## Permission Required

The browser always asks:

```
Allow this website to access your location?

[Allow]  [Block]
```

# 2. Drag and Drop API

Allows users to drag elements and drop them elsewhere.

Examples:

- Trello
- File uploads
- Kanban boards
- Reordering items

---

## How It Works

```
User drags item

      │

dragstart

      │

dragover

      │

drop

      │

Element moved
```

---

## Important Events

| Event | Description |
|--------|-------------|
| dragstart | Drag begins |
| drag | While dragging |
| dragenter | Enters target |
| dragover | Over target |
| dragleave | Leaves target |
| drop | Item dropped |
| dragend | Drag finished |

---

## Example

```html
<div draggable="true">
    Drag Me
</div>
```

# 3. Web Storage API

Stores data inside the browser.

Unlike variables, stored data remains available even after refreshing the page.

---

## Types

```
Web Storage

      │

 ┌───────────────┐
 │               │
 ▼               ▼

localStorage  sessionStorage
```

---

## localStorage

Data remains until removed manually.

```
Save today

↓

Refresh page

↓

Still available
```

Example:

```javascript
localStorage.setItem("name", "Navaneeth");

let name = localStorage.getItem("name");

localStorage.removeItem("name");

localStorage.clear();
```

---

## sessionStorage

Exists only while the browser tab is open.

```
Open tab

↓

Save data

↓

Close tab

↓

Data removed
```

Example:

```javascript
sessionStorage.setItem("theme", "dark");
```

---

## Comparison

| Feature | localStorage | sessionStorage |
|----------|--------------|----------------|
| Capacity | ~5–10 MB | ~5–10 MB |
| Lifetime | Permanent | Until tab closes |
| Shared between tabs | Yes (same origin) | No |

# 4. Web Workers API

JavaScript normally runs on a single thread.

Heavy tasks can freeze the webpage.

Web Workers solve this by running code in a background thread.

---

## Without Web Worker

```
JavaScript

↓

Heavy Calculation

↓

UI freezes

↓

Finished
```

---

## With Web Worker

```
Main Thread

        │

        ├──────── UI remains responsive

        │

Background Worker

↓

Heavy Calculation

↓

Result sent back
```

---

## Create Worker

```javascript
const worker = new Worker("worker.js");
```

---

## Send Data

```javascript
worker.postMessage(1000000);
```

---

## Receive Result

```javascript
worker.onmessage = function(event){
    console.log(event.data);
};
```

---

## Communication

```
Main JS

postMessage()

      │

      ▼

Worker

calculates

      │

postMessage()

      ▼

Main JS
```

# 5. Server-Sent Events (SSE)

SSE allows the **server** to continuously send updates to the browser over a single HTTP connection.

Ideal for:

- Live scores
- News feeds
- Notifications
- Stock prices
- Dashboards

---

## How It Works

```
Browser

        │

Single HTTP Connection

        │

Server

        │

Keeps sending updates

↓

Message 1

↓

Message 2

↓

Message 3
```

Unlike polling, the browser does **not** repeatedly ask for updates.

---

## Example

```javascript
const source = new EventSource("/events");

source.onmessage = function(event) {
    console.log(event.data);
};
```

---

## SSE vs Polling

### Polling

```
Browser

↓

Request

↓

Response

↓

Wait

↓

Request again

↓

Response
```

### SSE

```
Browser

↓

One Connection

↓

Server keeps sending updates
```

# 6. Fetch API

Used to communicate with servers.

Common uses:

- Login
- Load products
- Submit forms
- REST APIs

Example:

```javascript
fetch("/users")
    .then(response => response.json())
    .then(data => console.log(data));
```

---

# 7. Clipboard API

Read or write clipboard data.

Example:

```javascript
navigator.clipboard.writeText("Hello");
```

Read:

```javascript
navigator.clipboard.readText();
```

Common uses:

- Copy buttons
- Password managers
- Share links

---

# 8. Notifications API

Displays desktop notifications.

Example:

```javascript
Notification.requestPermission();
```

Then:

```javascript
new Notification("Order Delivered");
```

Requires user permission.

---

# 9. History API

Allows JavaScript to manipulate browser history without reloading the page.

Example:

```javascript
history.back();

history.forward();

history.pushState({}, "", "/profile");
```

Used heavily in Single Page Applications (SPAs).

---

# 10. Fullscreen API

Allows an element to enter fullscreen mode.

Example:

```javascript
document.documentElement.requestFullscreen();
```

Exit:

```javascript
document.exitFullscreen();
```

Used in:

- Video players
- Games
- Presentations

# Key Points

- Web APIs are provided by the **browser**, not JavaScript.
- Most Web APIs are asynchronous because they interact with external resources.
- Many APIs require user permission (such as Geolocation and Notifications).
- Web Storage allows data to persist across page reloads.
- Web Workers prevent the UI from freezing during heavy computations.
- SSE is designed for one-way, real-time updates from the server to the browser.
- The Fetch API is the modern way to communicate with web servers.
- Understanding these APIs is essential for building modern web applications.
