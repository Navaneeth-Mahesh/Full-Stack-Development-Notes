# Web Components

Web Components are a set of web platform features that allow you to create **reusable, custom HTML elements**.

They help you build modular, encapsulated UI components that work across frameworks.

---

# Main Features of Web Components

Web Components are built using 3 core technologies:

1. **Custom Elements**
2. **Shadow DOM**
3. **HTML Templates**

---

# 1. Custom Elements

Custom Elements allow you to define your own HTML tags.

Example:

```js
class MyButton extends HTMLElement {
    connectedCallback() {
        this.innerHTML = "Click Me";
    }
}

customElements.define("my-button", MyButton);
```

Usage:

```html
<my-button></my-button>
```

---

# 2. Shadow DOM

Shadow DOM provides **encapsulation** for HTML, CSS, and JavaScript.

It prevents styles from leaking in or out.

```js
class MyCard extends HTMLElement {
    constructor() {
        super();
        const shadow = this.attachShadow({ mode: "open" });

        shadow.innerHTML = `
            <style>
                p {
                    color: red;
                }
            </style>
            <p>This is inside Shadow DOM</p>
        `;
    }
}

customElements.define("my-card", MyCard);
```

---

# 3. HTML Templates

Templates define reusable HTML that is not rendered immediately.

```html
<template id="cardTemplate">
    <style>
        .card {
            border: 1px solid black;
            padding: 10px;
        }
    </style>

    <div class="card">
        <h3>Title</h3>
        <p>Description here</p>
    </div>
</template>
```

Using the template:

```js
const template = document.getElementById("cardTemplate");
const clone = template.content.cloneNode(true);
document.body.appendChild(clone);
```

# Lifecycle Callbacks

Custom elements have lifecycle methods:

| Method | Description |
|--------|-------------|
| `connectedCallback()` | Called when element is added to DOM |
| `disconnectedCallback()` | Called when element is removed |
| `attributeChangedCallback()` | Called when attributes change |

Example:

```js
class MyElement extends HTMLElement {
    connectedCallback() {
        console.log("Element added to page");
    }
}
```

---

# Attributes in Custom Elements

You can pass data using attributes.

```html
<user-card name="Navaneeth"></user-card>
```

```js
class UserCard extends HTMLElement {
    connectedCallback() {
        const name = this.getAttribute("name");
        this.innerHTML = `Hello ${name}`;
    }
}

customElements.define("user-card", UserCard);
```

---

# Advantages of Web Components

- Reusable across projects
- Framework independent
- Encapsulated styles and logic
- Native browser support

---

# Limitations

- Older browsers may need polyfills
- More boilerplate compared to frameworks
- Smaller ecosystem compared to React/Vue

---

# Summary

- Web Components let you create custom HTML elements.
- Built using Custom Elements, Shadow DOM, and Templates.
- They improve reusability and modularity in web development.
