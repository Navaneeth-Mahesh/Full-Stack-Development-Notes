# What is an Array?

An **Array** is one of the most fundamental data structures in Computer Science.

An array is a **collection of elements of the same data type stored together in contiguous (continuous) memory locations.**

In simple words,

> An array stores multiple values inside a single variable, where every value has its own position called an **index**.

Instead of creating many variables like this:

```java
int marks1 = 85;
int marks2 = 90;
int marks3 = 78;
int marks4 = 88;
int marks5 = 95;
```

We can store all values in one array:

```java
int[] marks = {85, 90, 78, 88, 95};
```

Now every mark has its own index.

```
Index :  0    1    2    3    4
Value : 85   90   78   88   95
```

---

# Simple Definition

> An array is a fixed-size collection of similar data stored one after another in memory.

---

# Technical Definition

An array is a linear data structure where

- all elements have the same data type
- memory is allocated continuously
- every element can be accessed directly using its index
- indexing starts from **0** in Java

---

# Why Do We Need Arrays?

Imagine a teacher storing marks of 100 students.

Without arrays:

```java
int student1;
int student2;
int student3;
...
int student100;
```

Problems:

❌ Too many variables

❌ Hard to manage

❌ Difficult to process

❌ Impossible to loop efficiently

Now using arrays:

```java
int[] marks = new int[100];
```

Only one variable stores all marks.

We can process every student using loops.

# Memory Representation

Arrays occupy **continuous memory**.

Suppose one integer occupies **4 bytes**.

Base Address = 1000

```
Address      Value

1000 -------- 5

1004 -------- 10

1008 -------- 15

1012 -------- 20

1016 -------- 25
```

Notice

Every next integer starts after 4 bytes.

This continuous storage makes arrays extremely fast.

# How Java Stores Arrays

When you write

```java
int[] arr = {10,20,30,40};
```

Java actually does two things.

### Step 1

Creates an array object inside Heap Memory.

```
Heap

---------------------
|10|20|30|40|
---------------------
```

### Step 2

Stores its reference inside variable `arr`.

```
Stack

arr
 |
 |
 V

Heap

---------------------
|10|20|30|40|
---------------------
```

So,

`arr` does **not** contain the actual values.

It contains the **reference (memory address)** of the array object.

# Why Does Indexing Start from 0?

This is one of the most common interview questions.

Suppose

```
Base Address = 1000
```

Formula:

```
Address = Base + (Index × Size of Data Type)
```

Example

```
arr[0]

Address

1000 + (0 × 4)

=

1000
```

```
arr[3]

Address

1000 + (3 × 4)

=

1012
```

Starting from 0 makes the calculation simple and efficient.

If indexing started from 1:

```
Base + ((Index-1) × Size)
```

Extra subtraction would be needed every time.

That's why almost all modern programming languages use **0-based indexing**.

# Advantages of Arrays

## 1. Fast Access

Access any element instantly.

```java
System.out.println(arr[4]);
```

Time Complexity:

```
O(1)
```

---

## 2. Easy Traversal

Loops make processing simple.

```java
for(int num : arr)
{
    System.out.println(num);
}
```

---

## 3. Memory Efficient

No extra node objects like Linked Lists.

Only actual data is stored.

---

## 4. Cache Friendly

Since data is continuous,

CPU cache works efficiently,

making arrays faster than many other data structures.

---

# Disadvantages of Arrays

## Fixed Size

Cannot grow automatically.

```java
int[] arr = new int[5];
```

Only five elements.

Need a bigger array to store more.

---

## Same Data Type

Allowed

```java
int[] numbers;
```

Not Allowed

```java
int[] data = {10, "Hello"};
```

Reason:

Arrays store homogeneous data.

---

## Insertion is Expensive

Adding in the middle requires shifting.

Example

Before

```
10 20 30 40
```

Insert 15

```
10 15 20 30 40
```

Everything shifts.

---

## Deletion is Expensive

Deleting also requires shifting.

```
10 20 30 40
```

Delete 20

```
10 30 40
```

Again shifting occurs.

# When Should We Use Arrays?

Use arrays when:

- Number of elements is known.
- Fast access is required.
- Elements are of the same type.
- Frequent searching by index is needed.
- Memory efficiency is important.

# Interview Questions

## Q1. What is an Array?

**Answer:**

An array is a linear data structure that stores multiple elements of the same data type in contiguous memory locations. Each element is accessed using an index.

---

## Q2. Why are arrays faster than Linked Lists?

**Answer:**

Arrays store data in continuous memory, allowing direct index-based access (`O(1)`) and better CPU cache utilization. Linked Lists require traversing nodes (`O(n)`) for indexed access.

---

## Q3. Why is array indexing zero-based?

**Answer:**

Because the memory address of an element is calculated using:

```
Address = Base Address + (Index × Size)
```

Starting from 0 eliminates the need for an extra subtraction, making access more efficient.

---

## Q4. Can an array store different data types?

**Answer:**

No. An array stores only elements of the same data type. For mixed data, Java provides classes, objects, or collections like `ArrayList<Object>`.

---

## Q5. What are the main limitations of arrays?

**Answer:**

- Fixed size
- Homogeneous elements only
- Costly insertion and deletion in the middle
- Memory size cannot be changed after creation

---

# Key Takeaways

- An array stores multiple values under one variable.
- All elements have the same data type.
- Elements are stored in contiguous memory.
- Every element has a unique index starting from `0`.
- Arrays provide `O(1)` random access using indices.
- Arrays have a fixed size after creation.
- They are one of the most important and widely used data structures in programming and interviews.
